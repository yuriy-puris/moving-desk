# Skill: Explore

## When to use
After analysis, before implementation.

## Read before starting
- Analyzer output
- Actual source files in scope (not just filenames)

## What to look for
- Existing patterns (route structure, service structure, hook structure)
- Reusable utilities and components
- Files that will be modified (not just created)
- Potential naming conflicts

## Output goes to
- Implementer agent as context

## Rules
- Read real files — never assume structure
- If codebase is empty — say so, skip pattern examples
- Keep output concise — map, not copy
